# 大阪カフェ出店タイプ別診断

## 概要

大阪府内のカフェ・スイーツ店舗データを分析し、エリアごとの店舗供給傾向を可視化するとともに、出店予定店舗の特徴から想定される店舗タイプを診断するWebアプリです。

都市部では「カフェ難民」と呼ばれる混雑が日常的に発生している一方、駅から離れたエリアでは店舗数そのものが少ないケースも見られます。
本プロジェクトでは以下の仮説をもとに、大阪府全域のカフェ供給状況を分析しました。

- 需要に対して供給が不足しているエリアが存在する
- エリア特性に対して店舗タイプが適切ではない

## データ

### 取得元

ホットペッパー グルメサーチ API

### 取得条件

| 項目 | 内容 |
|---|---|
| 対象エリア | 大阪府 |
| 対象ジャンル | カフェ・スイーツ |
| 取得件数 | 約1,150件（クレンジング後1,016件） |

### データの前提

ホットペッパーには国内大手カフェチェーン店（スターバックス・ドトール等）がほとんど含まれません。
**本データセットは個人店・中小チェーン中心**となっています。
なお本プロジェクトは取得時点のスナップショット分析であり、閉店・新規出店はデータに反映されません。

## 分析手法

### K-Meansクラスタリング

レビュー点数や売上といった目的変数が存在しないため、教師あり学習は採用できませんでした。
「大阪にはどのような店舗タイプが存在するのか」という現状把握を目的に、K-Meansクラスタリングを採用しています。
得られたクラスタを疑似的な目的変数として活用し、ユーザー入力から該当タイプを返すアプリへ発展させる構成としています。

### 使用特徴量

| カテゴリ | 特徴量 |
|---|---|
| 立地・規模 | 席数・駅からの徒歩時間・エリア供給密度 |
| 設備環境 | Wi-Fi・カード決済・駐車場・禁煙情報 |
| 利用ターゲット | 子供可 |
| 営業形態 | ランチ提供・平日定休日・土日定休日 |

### クラスタ数の決定

4〜6クラスタで比較検証を行い、以下の条件をすべて満たす**5クラスタ**を採用しました。

- 各クラスタの店舗数が100件以上
- クラスタ間の特徴差が明確
- 人が解釈・命名しやすい

## クラスタリング結果

| クラスタ | 店舗タイプ | 件数 | 特徴 |
|---|---|---|---|
| 0 | 駅近ランチカフェ | 265件 | 駅近・ランチ中心・Wi-Fi/カード決済対応率高 |
| 1 | 大衆向けコンセプトカフェ | 242件 | 子供連れ可・カード決済非対応率高 |
| 2 | 大人向けコンセプトカフェ | 194件 | 子供可0.00で完全分離・コーヒー専門店等 |
| 3 | 郊外ランチカフェ | 117件 | 駐車場完備・徒歩時間突出・ロードサイド型 |
| 4 | 都市部大規模カフェ | 198件 | 席数多・都心集中・ほぼ毎日営業 |

## アプリ機能

### 入力項目

| カテゴリ | 項目 |
|---|---|
| 基本情報 | 出店エリア・席数・駅からの徒歩時間 |
| 設備 | Wi-Fi・カード決済・駐車場 |
| 利用ターゲット | 禁煙情報・お子様連れ |
| 営業スタイル | 平日定休日・土日定休日・ランチメニュー提供 |

### 出力内容

- 想定店舗タイプの診断結果
- 該当タイプの特徴・代表エリアの説明
- 出店戦略ポイント
- エリア内店舗タイプ構成比（棒グラフ）
- 同タイプ店舗割合と競合環境のコメント

## 工夫した点

**① 自由記述データの数値化**
アクセス情報から徒歩時間、営業時間から定休日を正規表現で抽出しました。
特に徒歩時間は「徒歩◯分」「◯m」「◯km」「すぐ・直結」など複数の表記パターンに対応しています。
なお歩行速度は不動産広告のルールとして定められている分速80mを参考にしました。

**② エリア特性の数値化**
エリア名をそのまま特徴量にするとカテゴリ数が増えすぎるため、エリア内店舗数をエリア供給密度として数値化しました。

**③ 解釈可能性を重視したクラスタリング**
分類精度だけでなく「人が理解できるクラスタ」となるよう、特徴量の採否・クラスタ数・命名を繰り返し検討しました。
